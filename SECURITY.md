# Security Policy

## Unterstützte Versionen
Aktuell werden Sicherheits-Updates nur für die neueste Version im `main`-Branch bereitgestellt.

| Version | Unterstützt |
| ------- | ----------- |
| > 1.0.0 | ✅ Ja       |
| < 1.0.0 | ❌ Nein     |

## Schwachstellen melden
Ich nehme die Sicherheit dieses Cloud-Services sehr ernst. Wenn du eine Sicherheitslücke entdeckt hast, gehe bitte wie folgt vor:

1. **Nicht öffentlich machen:** Bitte erstelle kein öffentliches Issue für Sicherheitslücken.
2. **Private Meldung:** Sende eine detaillierte E-Mail an [deine-email@beispiel.de].
3. **Reaktionszeit:** Ich werde versuchen, innerhalb von 48 Stunden auf die Meldung zu reagieren und einen Zeitplan für die Behebung zu erstellen.

## Grundsätze
Dieses Projekt nutzt automatisierte Tools wie GitHub Secret Scanning und Dependabot. Sollten Zugangsdaten (Secrets) im Verlauf des Projekts versehentlich veröffentlicht worden sein, werden diese sofort in der AWS-Konsole widerrufen und ersetzt.
